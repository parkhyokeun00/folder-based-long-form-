# Voice Patterns

This page stores reusable narration rules for the host voice.

## Rule 1. Sound guided, not neutral

- speak like a host who is actively taking the viewer somewhere
- explanation alone is not enough

## Rule 2. Translate abstraction fast

- if a concept name appears, convert it into viewer language quickly
- do not leave theory words hanging without a plain-language bridge

## Rule 3. Let the host be audible

- keep at least one host-voice sentence in each major section
- examples:
- `이상하죠`
- `여기서부터가 재밌습니다`
- `핵심은 여기서 갈립니다`
- caution: these lines should add pressure or interpretation, not tell the viewer what reaction to have

## Rule 4. Use questions as pull, not filler

- question tags are useful when they move the viewer deeper
- weak version: one-line empty question
- strong version: scene -> mismatch -> question -> answer pressure

## Rule 5. Protect spoken rhythm

- write for a mouth, not for a page
- when a paragraph feels like a report, cut or split it

## Rule 6. Do not confuse compression with clarity

- short is not automatically strong
- if trimming removes actors, responses, counter-claims, or consequences, the script is underwritten, not efficient
- trim repetition after the argument is complete, not before
