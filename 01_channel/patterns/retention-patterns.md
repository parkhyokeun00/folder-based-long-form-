# Retention Patterns

This page stores reusable middle and ending structures that improve watchability.

## Pattern 1. Name the pressure, not just the background

- history works better when it explains what pressure made the behavior useful
- good pressure families:
- status
- survival
- commerce
- privacy
- crowding
- reputation

## Pattern 2. Middle turns must change the meaning

- the middle should not only add facts
- it should reframe the topic at least once
- default move: stereotype -> hidden logic -> historical usefulness

## Pattern 3. Modern re-entry

- do not end a culture topic inside the past
- return to how the pattern survives now:
- weakened habit
- meme
- stereotype
- commodity
- identity marker

## Pattern 4. Ending residue

- a strong ending changes how the opening is remembered
- the best endings either compress the hidden mechanism in one line or open the next question cleanly
