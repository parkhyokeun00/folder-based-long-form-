## Benchmark translation

- keep the opening and closing function stable
- start from a strange case, not a thesis
- when the topic supports it, open with a chain of examples rather than one polite sample
- convert abstract language into everyday viewer language
- use soft question tags to keep psychological proximity
- let the host react out loud instead of hiding behind neutral explanation
- drive culture topics through origin, spread, and modern afterlife
- build an episode network, not isolated uploads

## Callback and network rules

- every episode should check for one backward callback
- every episode should check for one forward seed
- only use callbacks that add meaning, not filler

## Retention hypotheses

- strongest opening shape: strange case -> rapid example cluster -> `그런데` -> hidden mechanism
- weakest middle pattern: historical explanation that is correct but detached from survival, class, or daily inconvenience
- analogy style that feels alive: polite surface versus hidden line, smile versus rejection, hospitality versus boundary
- signs the draft sounds like a lecture: too few examples, no host reaction, no present-day return, definitions arriving before curiosity

## Permanent craft rules

- prove the pattern fast before naming it academically
- treat examples as entertainment and evidence at the same time
- every long middle section should answer `why was this useful then`
- every ending should answer `what form does this take now`

## Recursive engine rules

- the engine repo stores generalized rules, not raw experiment clutter
- result workspaces may contain honest failure notes, but engine promotions must be rewritten as reusable positive law
- when a lesson needs more proof, hold it locally instead of forcing it into the engine
- use pattern pages for mid-level learnings and reserve this file for the highest-level channel law

## Postmortem format

After each upload, record:

- what the thumbnail promised
- what the first minute proved
- where the middle might sag
- which line or analogy deserves reuse
- which lesson should become a permanent rule
