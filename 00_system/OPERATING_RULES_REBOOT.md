# Operating Rules Reboot

This document supersedes older workflow assumptions in this workspace when they conflict.

The old system over-weighted:

- hook-first locking
- thumbnail-first packaging
- draft-to-final compression as a default move
- structural neatness before case coverage

This reboot rejects those defaults.

## Core stance

- Research comes before packaging.
- Cases come before summaries.
- A script earns its title and thumbnail after its core claim survives drafting.
- Length should come from real material, not from repeated conclusions.
- Compression is a late editorial option, not the primary writing mode.

## New sequence

1. Define the real question.
2. Collect cases broadly.
3. Separate facts, examples, interpretations, and hypotheses.
4. Decide the central claim only after the case field is visible.
5. Draft long enough to discover the real structure.
6. Cut repetition after discovery, not before.
7. Package last: title, thumbnail, opening promise, publish card.

## Case requirements

- Do not build from one explainer article alone.
- For culture, language, and internet-history topics, gather either:
- at least 5 independently useful cases
- or 3 deep cases plus 2 context sources that change the frame
- Each case must do a different job.
- Before drafting, state what each case proves.
- If multiple cases only restate the same conclusion, merge or drop them.
- When possible, use one anchor case, then 2 to 3 support cases instead of many rival centers.
- For concept-heavy topics, include at least one adjacent concept contrast set.

## Drafting rules

- Do not lock thumbnail or hook before the draft has real evidence behind it.
- Do not compress the draft into the final script by default.
- The final script may be longer than the draft if coverage is still thin.
- Do not force a universal frame too early.
- Let the material show what the center is before you name the center.
- One piece should have one governing claim.
- Side insights may stay, but they must serve the governing claim.
- Open from a familiar present-day anomaly before diving into origin when that route sharpens the question.

## Repetition rules

- A conclusion may be stated once cleanly and once reframed if necessary.
- If a paragraph only rephrases what a previous paragraph already proved, cut it.
- End once.
- Do not build length by escalating diction around the same idea.

## Packaging rules

- `01_thumbnail-hook.md` is now a late-stage packaging memo, not a gate.
- Packaging is provisional until the script spine is stable.
- Thumbnail/title should surface the real pressure of the piece, not a guessed pressure.
- If the finished script changes the center, packaging must change with it.

## Source discipline

- Keep `raw`, `parsed`, and `canon-integrated` notes distinct.
- Mark uncertain expansion paths as hypotheses, not facts.
- Do not borrow authority from a source that did not actually prove your claim.
- Use secondary commentary to widen context, not to replace evidence.

## Review pass

Before finalizing, check:

- what is the single governing claim
- which case is the anchor case
- what each supporting case uniquely contributes
- whether the ending is already complete one paragraph earlier
- whether any abstract phrase is covering missing research

## Anti-patterns

- locking hook/title before the material exists
- summarizing before the case field is built
- padding with conclusions
- compressing because the material is thin
- inventing nuance instead of researching nuance
- turning critique into scripture
